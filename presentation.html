Motivation

Why measure biosecurity risk in LLMs?

  • Open-weight models are freely downloadable, fine-tunable, and uncontrolled — safety alignment can be stripped out post-download
  • 2025 novice uplift studies: LLMs now raise novice biology performance to expert baseline on dual-use tasks (virology troubleshooting, molecular cloning)
  • VCT benchmark: OpenAI o3 outperforms 94% of expert virologists on virology knowledge — frontier models already exceed most specialists
  • The WMDP paper (Li et al., ICML 2024) introduced the benchmark and RMU unlearning method — we build directly on this work
  • Our finding: Falcon3-7B scores 70.9% on WMDP-bio — far above the 25% random baseline, approaching frontier at the <10B parameter scale
Research Gap — Filled

We filled the Falcon3 biosecurity gap

  • Falcon3 (TII, Dec 2024) = #1 ranked open-weight family on HuggingFace <13B leaderboard — widely used, widely deployed
  • Llama, Mistral, Qwen, DeepSeek all had published WMDP scores. Falcon3 had none.
  • We ran all 4 Falcon3 sizes (1B → 3B → 7B → 10B) on the full WMDP-bio test set (n=1,273 each) — first published evaluation of this family
  • Also benchmarked 5 size-matched models (Llama3.1-8B, Qwen2.5-7B, Mistral-7B, Phi4-mini-3.8B) for peer comparison at the 7–8B tier
Key Result

Strong monotonic scaling from 40.1% (1B) to 73.7% (10B). Falcon3-7B (70.9%) is competitive with same-size SOTA. System prompt framing has no effect on demonstrated capability. RMU unlearning is the immediate next step.

Null Result

All 95% CIs overlap completely. System prompts have no statistically significant effect on WMDP-bio accuracy. Parametric biosecurity knowledge is robust to surface-level instructional framing — you cannot reduce demonstrated capability by changing the prompt.

Strong Negative: −28.0pp

Valid CoT (512 tok) drops accuracy from 70.9% → 42.9%. Model reasons into wrong answers despite full token budget. 32-tok run is a truncation artifact (250 format failures confirm model was cut off mid-thought). Also 14× slower: 84.5 min vs 6.1 min.

Methods

Three critical protocol lessons

  • Chat template collision — Inspect's hf/ provider AND Falcon3's tokenizer both applied instruction formatting → double-wrapped prompts → 5–10% accuracy (well below random chance)
    Fix: use ollama/ provider exclusively. Ollama handles the chat template internally. Always verify format_error rate before trusting accuracy numbers.
  • Scoring method gap — Li et al. 2024 used logprob scoring (probability over the four answer tokens). We used text-gen + regex extraction. Text-gen scores are generally lower than logprob scores and are not directly comparable. Our 70.9% ≠ Li et al.'s numbers — different protocol.
    Interpret results within our cohort only (Falcon3 vs same-protocol baselines).
  • CoT token budget — Setting max_tokens=32 with chain_of_thought() silently truncates reasoning mid-sentence. The model never outputs a final answer letter → 19.6% format failures → near-random 29.4% score. Always match token budget to prompt type. This is not visible from accuracy alone — check format_error rate.
Protocol hygiene

Anomalous results (below random, high format failures) almost always have a methodological explanation. Pilot at n=10, inspect raw outputs with inspect view, verify format_error rate before committing to 1,273-sample runs.

Machine Unlearning

RMU: Representation Misdirection for Unlearning

The method (Li et al. 2024)
  • Forget loss: on hazardous data — push hidden states toward a random noise vector r (misdirect, not suppress)
  • Retain loss: on benign data — keep hidden states close to the original model's representations
  • Total loss: L = L_forget + α · L_retain where α balances forgetting vs. retention
  • Operates on a single intermediate layer · few training epochs · fast
Unlearning ≠ Refusal Training

Refusal adds a behavior-level block — circumventable by fine-tuning or jailbreaking. RMU degrades parametric knowledge at the representation level — harder to reverse. Our ablation showed system prompts can't touch the underlying capability; RMU operates one level deeper.

Next Steps

RMU execution plan for Falcon3-7B

Execution steps
  • Step 0 — Retain baseline: run MMLU on Falcon3-7B before RMU to establish the preserve metric
  • Step 1 — Setup: clone WMDP repo (github.com/centerforaisafety/wmdp) · prepare forget set (bio corpora) + retain set (Wikitext / MMLU general)
  • Step 2 — Train RMU: apply forget+retain loss on Falcon3-7B · target intermediate layer · few epochs · α calibration
  • Step 3 — Re-evaluate: WMDP-bio (n=1273) + MMLU on unlearned model · same protocol as baseline
  • Step 4 — Report: before/after delta table + MMLU preservation check
Open questions & risks
  • Can M2 handle 7B backward pass? Inference works; training (gradient checkpointing likely needed) is heavier
  • Which layer to target? WMDP paper used layer 7 of Llama 2-7B — Falcon3 architecture may differ
  • What α (retain weight) transfers? Paper: α=1200 for bio — needs calibration for Falcon3
  • Will 70.9% drop cleanly to ~25%? Pilot at n=200 first to check direction before full sweep
Fallback plan

If M2 can't run 7B RMU in reasonable time → fall back to Falcon3-3B (57.9% → ~25%). Smaller backward pass, same method, same story. The RMU demonstration is what matters, not the specific model size.
